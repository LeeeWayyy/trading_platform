# ADDING A NEW DEPENDENCY:
# ===========================================================================
#   1. Identify the high-level package needed
#   2. Install: pip install <package>
#   3. Regenerate with header preserved:
#      head -51 requirements.txt > /tmp/req_header.txt
#      pip freeze > /tmp/req_body.txt
#      cat /tmp/req_header.txt /tmp/req_body.txt > requirements.txt
#   4. Update the "HIGH-LEVEL DEPENDENCIES" section above
#   5. Commit with clear description of what high-level dep was added
#
# ===========================================================================
# RATIONALE FOR LOCKFILE APPROACH:
# ===========================================================================
# Pros:
#   ✅ Reproducible builds (same versions across all environments)
#   ✅ Security (pin vulnerable packages to patched versions)
#   ✅ CI cache hits (dependencies don't change between runs)
#
# Cons:
#   ❌ Large diffs for dependency updates (100s of lines)
#   ❌ Merge conflicts on concurrent PRs touching dependencies
#   ❌ Harder to see which packages were intentionally added
